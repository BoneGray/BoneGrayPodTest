# TileSet 墙体碰撞规范

## 目标

墙体瓦块的碰撞不要只靠手拖，也不要只靠单块图片猜测。优先根据 Godot Terrain 的连接规则确定碰撞形态，再用像素边界微调透明区域。

## 基本规则

- 使用 Terrain 自动连接的墙体瓦块，优先读取 `terrains_peering_bit`。
- `left_side` 或 `right_side` 有连接时，对应碰撞边界必须贴到 tile 边缘，避免相邻墙块之间产生可卡住的缝。
- 没有连接的一侧，碰撞边界按该瓦块在有效碰撞高度内的非透明像素收缩，避免玩家被透明区域挡住。
- 横向墙体默认使用中下部阻挡带。
- 有 `bottom_side` 连接的竖向墙体、柱体或向下延伸墙块，碰撞底边应延伸到 tile 底部。
- 柱头、柱帽、石墩头、围栏端点等上半部实体可以单独生成碰撞，但必须避开透明区。
- 一个瓦块可以有多个碰撞矩形：横梁、竖柱、柱头应分开建模，避免用一个大矩形填掉中间空洞。
- 透明像素扫描用于微调没有 Terrain 连接的一侧，也用于收窄上半部特殊碰撞；连接方向始终以 Terrain 规则为准。

## Wall 3 当前标准

`Wall 3` 使用 32x32 tile，当前碰撞标准如下：

- 碰撞顶边：`y = -6`
- 横向墙体或没有向下连接的墙块：底边 `y = 10`
- 有 `bottom_side` 连接的竖墙、柱体、向下延伸墙块：底边 `y = 16`
- 有 `left_side` 连接：左边 `x = -16`
- 有 `right_side` 连接：右边 `x = 16`
- 无左右连接的一侧：按该高度带内非透明像素边界收缩
- 有明显柱头/上半部实体：额外生成一块上半部碰撞，但只覆盖实体核心，透明角落必须放出来
- 拐角、T 字块、柱体端点允许同时存在 2 到 3 个 `polygon`：横梁、竖柱、柱头分别建模

示例：

```text
右侧连接、左侧透明：
PackedVector2Array(-5, -6, 16, -6, 16, 16, -5, 16)

左右都连接：
PackedVector2Array(-16, -6, 16, -6, 16, 10, -16, 10)

竖向独立柱体：
PackedVector2Array(-5, -6, 5, -6, 5, 16, -5, 16)
```

## 后续处理其它墙体

处理 `Wall 0`、`Wall 1`、`Wall 2` 或其它围栏时，先按同样规则生成碰撞模板，再进游戏测试手感。不要直接复制 `Wall 3` 的具体数值，除非它们的视觉尺寸和脚底阻挡位置一致。
